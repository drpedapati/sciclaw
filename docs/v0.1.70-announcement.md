## Tweet 2 — Edit-to-mention

You're in a Discord channel writing a detailed request to your paired scientist. You hit send. Realize you forgot the @mention. The bot ignores it.

Before: retype the whole thing.
Now: just edit the message and add the @mention. sciClaw sees the edit and responds.

Typo-fix edits without a mention are still ignored — so it won't double-fire if you're just fixing spelling.

Also new: the bot now responds when you reply to one of its messages, and when you @mention a role the bot belongs to.

## Tweet 3 — Discord archive recall

sciClaw now archives Discord conversations and can recall them during the agent loop.

When you ask "what did we discuss about the analysis last week?" the agent searches its archive of your channel history and brings relevant context into the conversation — without you having to scroll back and copy-paste.

Archive commands:
```
sciclaw archive discord list
sciclaw archive discord run
sciclaw archive discord recall "search term"
```

## Tweet 4 — Cloud workspace reliability

If your routed workspaces live on Dropbox, iCloud, or another cloud-synced folder, previous versions could stall when the filesystem was slow to respond.

v0.1.70 makes everything fail-open:
- State bootstrap, session preload, hook audits, and archive checks all have bounded timeouts
- Filesystem read/list tools won't hang the agent if a file takes too long
- State persistence is async — a slow Dropbox sync won't block your next message

This matters if you're running sciClaw on a shared lab server with cloud-backed project folders.

## Tweet 5 — Getting started

New to sciClaw?

```
brew tap drpedapati/tap && brew install sciclaw
sciclaw app
```

The setup wizard walks you through connecting your AI provider and pairing a chat channel. Takes about 2 minutes.

Already on sciClaw?

```
brew update && brew upgrade sciclaw
sciclaw doctor
```

No data leaves your machine. Your paired scientist lives on your computer.
