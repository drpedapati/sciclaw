# sciClaw v0.1.73 Announcement

## Tweet 1 - Main Announcement

sciClaw v0.1.73 is out.

New channel_history tool — the bot can now read actual Discord channel messages, not just its own session memory. Also: per-turn token usage logging and a fix for gateway.log never writing structured JSON.

https://github.com/drpedapati/sciclaw/releases/tag/v0.1.73

---

## Tweet 2 - Channel History

Before v0.1.73, asking sciClaw to "summarize this channel" would only show messages from its own session file — missing everything else in the channel.

Now it calls the Discord API directly, fetches up to 100 messages, and gives you a real summary. Works across all users and messages, not just ones directed at the bot.

---

## Tweet 3 - Token Visibility

Every LLM call now logs input/output token counts. Per-turn summaries show total tokens, number of LLM calls, and duration.

A simple "status" command: ~13k tokens (most of it system prompt + tool definitions). A channel summary with 100 messages: ~26k tokens. Now you can see exactly where the cost goes.

---

## Tweet 4 - Upgrade

```bash
brew update && brew upgrade sciclaw
sciclaw service restart
```

Check your token usage:
```bash
grep "Turn token usage" ~/.picoclaw/gateway.log | tail -5
```
